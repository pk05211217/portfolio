# 個人作品集網站規劃書 (Portfolio Website Specifications & Plan)

本規劃書旨在定義個人作品集網站的定位、功能架構、視覺風格、技術選型及開發時程，作為後續設計與開發的指導依據。

---

## 1. 專案概述與目標 (Project Overview & Objectives)

- **網站名稱：** [您的名字/品牌名稱] 個人作品集網站
- **專案定位：** 展現個人專業技能、精選專案成果與個人特質的數位名片。
- **核心目標：**
  1.  **提升專業形象：** 透過高品質的視覺與流暢的互動，建立值得信賴的專業品牌。
  2.  **吸引潛在機會：** 吸引獵頭、HR、自由接案客戶（Freelance Clients）或合作夥伴主動聯繫。
  3.  **整合個人資產：** 將分散在 GitHub、Behance、LinkedIn、部落格等平台的成果集中管理與呈現。

## 3. 網站架構與內容規劃 (Information Architecture)

本網站採 **響應式單頁式（Single Page Application / Scroll Page）搭配獨立專案內頁** 設計，確保良好的瀏覽體驗。

```
[首頁 Home]
 ├── 1. 英雄總覽區 (Hero Section) -> 強烈的第一印象與一句話定位
 ├── 2. 關於我 (About Me) -> 簡介、核心優勢、特質
 ├── 3. 核心技能 (Skills) -> 技術棧、專業領域工具
 ├── 4. 精選作品 (Portfolio/Projects) -> 網格化作品展示 (點擊進入內頁)
 ├── 5. 工作經歷/里程碑 (Timeline) -> 學經歷時間軸
 └── 6. 聯絡我 (Contact) -> 表單與社群連結
```

### 各區塊詳細內容：

1.  **英雄總覽區 (Hero Section)：**
    - 主標題：一句話定義你是誰、你能解決什麼問題（例：_「我是 XXX，專注於打造流暢使用者體驗的前端工程師」_）。
    - 行動呼籲（CTA）：[瀏覽作品] 或 [與我聯絡] 按鈕。
    - 背景：簡約質感的幾何圖形、動態微交互或個人專業形象照。
2.  **關於我 (About Me)：**
    - 文字簡介：200-300 字的個人故事、熱情所在與專業價值。
    - 標籤/關鍵字：例如 #前端開發 #UI設計 #問題解決者。
3.  **核心技能 (Skills)：**
    - 分類呈現：前端技術（HTML/CSS/JS/React）、後端（Node.js/Python）、工具（Git/Figma）等。
    - _註：避免使用百分比條（如 JavaScript 80%），改用熟練度文字說明或標籤。_
4.  **精選作品 (Portfolio/Projects)：**
    - 以卡片式（Card Layout）呈現，每項作品包含：專案圖片、專案名稱、簡短摘要、使用技術標籤。
    - 點擊後導向 **專案詳情頁 (Case Study)**：
      - 專案背景與挑戰（Background & Challenge）
      - 你的角色與職責（Role）
      - 解決方案與技術實作（Solution & Execution）
      - 成果與量化效益（Results - 例如：提升 20% 載入速度）
      - 相關連結：[線上展示 Live Demo]、[原始碼 GitHub]
5.  **工作經歷與學歷 (Experience/Timeline)：**
    - 採用倒敘式的時間軸（Timeline），清晰呈現過往任職公司、職稱與核心成就。
6.  **聯絡我 (Contact)：**
    - 聯絡表單：姓名、電子郵件、主要主旨、留言內容。
    - 直接聯絡管道：Email 直寄連結。
    - 社群圖標：GitHub, LinkedIn, Medium, Behance 等。

## 4. 視覺與設計風格 (Visual & Design Direction)

- **設計風格：** 極簡主義 (Minimalism) / 現代科技感 / 專業洗鍊風。以大膽的留白（White Space）與精緻的字體排版（Typography）為主。
- **色彩計畫 (Color Palette)：**
  - **主色（60%）：** 潔淨白 / 深邃黑（支援 Dark Mode 暗黑模式切換）。
  - **輔助色（30%）：** 沉穩的高級灰 / 莫蘭迪色系，用於區塊背景與次要文字。
  - **點綴色（10%）：** 活力藍 / 科技綠 / 亮眼橘，用於 CTA 按鈕、連結、重點標示。
- **字體選用：**
  - 英文字體：Inter / Roboto / Montserrat (現代且具備極佳的可讀性)。
  - 中文字體：思源黑體 (Noto Sans TC) / 微軟正黑體。

## 5. 技術選型與架構 (Technology Stack)

根據開發成本與自訂需求度，建議採用以下方案之一：

| 方案 | 技術棧 (Tech Stack) | 優點 | 缺點 | 適用情境 |
| :--- | :------------------ | :--- | :--- | :------- |

| **方案 A：
純手工自建** | React / Next.js / Vue.js + Tailwind CSS | 自訂度 100%、SEO 表現極佳、效能優異、最能展現工程師實力。 | 開發時間較長、需自行處理部署。 | **前端/全端工程師** 首選，網站本身就是最強作品。 |
| **方案 B：
靜態網站生成** | Astro / Gatsby + Markdown | 載入速度極快、維護成本極低、適合整合部落格文章。 | 動態互動功能受限。 | **文字工作者、喜歡寫技術筆記的工程師。** |
| **方案 C：
無程式碼工具** | Webflow / Framer | 設計自由度極高、視覺動態強大、開發速度極快。 | 每月有平台維護費用、較難深度客製複雜後端功能。 | **UI/UX 設計師、產品經理、視覺設計師。** |

- **網域與部署 (Hosting & Domain)：**
  - 託管平台：Vercel / Netlify / GitHub Pages (皆有免費額度且支援 CI/CD 自動部署)。
  - 網域：建議購買個人專屬網域 (如 `yourname.com` 或 `yourname.dev`)，大幅提升專業感。

## 6. 接案與SEO最佳化 (SEO & Performance)

為了讓潛在客戶與獵頭更容易在 Google 搜尋到你：

- **SEO 關鍵字佈局：** 在網站標題（Title）與描述（Description）中加入：`[城市] + [專業職稱] + [您的名字]`（例如：_台北前端網頁工程師 - 王小明_）。
- **語意化標籤：** 嚴格使用 `&lt;header&gt;`, `&lt;main&gt;`, `&lt;section&gt;`, `&lt;footer&gt;`，以及正確的 `&lt;h1&gt;`-`&lt;h6&gt;` 層級。
- **圖片最佳化：** 所有作品截圖需經過壓縮，並優先使用 `.webp` 格式，且必須加上 `alt` 屬性說明。
- **響應式設計 (RWD)：** 確保行動裝置（手機、平板）的排版完美，因為超過 50% 的 HR 可能會先用手機點開你的履歷。

## 7. 開發時程與推進計畫 (Timeline & Roadmap)

預估總時程：**4 週** (以兼職/工作之餘開發為例)

- **Week 1：內容整理與文案撰寫**
  - 篩選並整理出最精選的 3-5 個作品，撰寫 Case Study 文案。
  - 準備好個人照片、個人簡介、更新最新版履歷。
- **Week 2：UI 設計與原型製作 (或挑選模板)**
  - 使用 Figma 進行視覺風格探索、定調顏色與字體。
  - 繪製高保真度設計圖（包含 Desktop & Mobile 版面）。
- **Week 3：程式碼開發 / 平台建置**
  - 建置前端環境，依據設計圖切版。
  - 實作響應式佈局與微交互動畫（如 RWD 選單、按鈕 Hover 效果）。
- **Week 4：測試、最佳化與正式上線**
  - 各瀏覽器相容性測試、手機實機測試。
  - 使用 Lighthouse 進行效能與 SEO 評測並進行最佳化。
  - 綁定個人網域、正式部署上線，並更新至 LinkedIn 等社群平台。
